In December, GitHub was the 7th-largest CVE issuer. By January it was number one,
and it has not given up the top spot since.

This is the monthly rank of the top CVE Numbering Authorities. Rank 1 means the
most CVEs published that month. Watch the red line climb.

The takeaway is not that GitHub ships buggier code. It is that the CVE system has
shifted. Platforms and ecosystems, GitHub Security Advisories, VulnCheck,
Patchstack, Wordfence, now assign a huge share of all CVEs, because they became
CNAs and started cataloging at scale. The old picture of a handful of big vendors
owning the list is gone.

If your mental model of "who reports vulnerabilities" is still Microsoft, Oracle,
and Adobe, the data has moved on.
